ODS App's Scripts tool allows Users to create and run server-side JavaScript scripts. These scripts run completely on the server, with only the result returned to the browser.



To create your script, there are two different cells that need to be filled. The first cell will contain the JavaScript code you want to run when your script is ran. The second cell will contain a JSON object of parameters that your script needs to be passed in. Once your script is ready, you can save your script by pressing the save script button at the bottom of the page.



Server Scripts can be ran either in the browser, or by calling the script API without the need for a browser. To run your script in the browser, press the Run Script button at the bottom of the page. There are several different ways that you can view the results of your query in the browser, which you can change by selecting the dropdown at the bottom of the page next to 'Results'. To run your script by calling the script's API, refer to the Script endpoint section at the bottom of your script's page. There you will find the endpoint along with other information about the API call.



There are two different Script pages, Server Scripts, and Admin Scripts. There are two main differences between these two pages. First of all, Server Scripts can be accessed by all users, whereas Admin Scripts can only be accessed by admins. Secondly, Admin Scripts can be run on a schedule, whereas Server Scripts can only be run from the browser or from an API call. To run an Admin Script on a schedule, press either save script or update script at the bottom of the page, where you can set the schedule that you want the script to run on from the dropdown options.



For information on additional methods available for use in Scripts, press the Script Help button on the right side of the page.
